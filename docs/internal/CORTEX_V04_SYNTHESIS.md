## DECISIONS (Step 5) — every ambiguity resolved

### D1: Tool count strategy
ORACLE-VISION says "don't add tools, extend existing ones."
ADVERSARY says "deprecate seek in favor of search."
**DECISION:** Create `m1nd.search` with 3 modes (literal/regex/semantic). Keep `seek` as alias that calls `search(mode="semantic")`. Net new tools: search, help, panoramic, efficiency = 4 new. Fix perspective = 0 new. Total: 56 → 60.

### D2: Personality in responses
Max said: "o m1nd tem que ter personalidade, identidade visual, calor nas respostas."
**DECISION:** Every tool response includes an optional `_m1nd` metadata field:
```json
{
  "result": { ... },
  "_m1nd": {
    "suggest_next": ["impact(node_id) to check blast radius", "learn(feedback) to strengthen edges"],
    "savings": {"tokens_saved": 750, "session_total": 48230},
    "gaia": {"global_tokens_never_burned": 2100000},
    "personality": "found 8 results for 'rate limiting'. top hit is smart_router.py (0.98) — 61 dependents, be careful here."
  }
}
```
The `personality` field is a one-line human-readable summary. NOT generated by LLM — template-based in Rust. Pattern: "{count} results for '{query}'. top hit is {label} ({score}) — {context_hint}."

### D3: Visual identity in help output
4 letters = 4 dimensions. This is the DNA.
```
M = #60A5FA (blue)     → STRUCTURAL (who calls who)
1 = #FBBF24 (gold)     → TEMPORAL (what changed together)

D = #818CF8 (indigo)   → SEMANTIC (naming patterns)
```

### D4: Gaia protection counter
Max said: "quanto o m1nd ajudou o planeta evitando que as llms fizessem leituras desnecessarias"
**DECISION:** Every response includes `_m1nd.gaia.global_tokens_never_burned`. Persists in `savings_state.json` alongside graph. Three levels:
- **query**: this specific query saved ~X tokens
- **session**: cumulative this session
- **global**: cumulative across all sessions (persisted)

Estimation formula:
- `activate/seek` → saved ~15 grep calls × ~50 tokens = 750 tokens
- `surgical_context` → saved ~8 Read calls × ~400 tokens = 3200 tokens
- `surgical_context_v2` → saved ~12 Read calls × ~400 tokens = 4800 tokens
- `impact/hypothesize/missing` → saved ~5 analysis calls × ~200 tokens = 1000 tokens
- `apply/apply_batch` → saved ~3 Write+verify calls × ~300 tokens = 900 tokens

### D5: suggest_next in EVERY response
Not just in help. EVERY tool response has `_m1nd.suggest_next[]`:
- After `activate` → suggest `impact(top_result)`, `learn(feedback)`, `hypothesize(claim)`
- After `impact` → suggest `validate_plan(files)`, `counterfactual(node)`
- After `hypothesize` → suggest `missing(query)`, `trace(error)`, `learn(feedback)`
- After `surgical_context` → suggest `apply(file, content)`, `apply_batch(edits)`
- After `apply` → suggest `predict(changed_node)`, `learn(feedback)`
Hardcoded mapping in Rust. ~20 lines.

### D6: report format
ORACLE-VISION says "integrate into trail.save."
**DECISION:** Keep `report` as standalone tool BUT also auto-generate mini-report in trail.save. Report returns markdown by default — optimized for LLM context injection.

### D7: panoramic weights
Default weights for combined_risk:
```
trust: 0.25
tremor: 0.20
layer_violations: 0.15
turbulence: 0.20
antibody_matches: 0.15
epidemic_risk: 0.05
```
Critical threshold: combined_risk >= 0.7

### D8: perspective.routes fix
Root cause (from ADVERSARY): `perspective.start` creates perspective with `focus_node: None` and never computes routes on cache miss.
**FIX:** After creating perspective, if focus_node resolved, compute routes immediately. If routes cache is None on `.routes()` call, recompute instead of returning empty.

### D9: efficiency honesty
ORACLE-VISION says: "be honest when grep would have been faster."
**DECISION:** If the query is a simple literal match on a known keyword, `_m1nd.savings.note` says "for this specific query, grep would be ~equal speed. m1nd's value is in the graph context, not raw speed."

### D10: search inverted index
ADVERSARY warns about ReDoS and unbounded results.
ORACLE-VISION suggests ripgrep integration.
**DECISION:** For literal/regex mode, build a simple inverted token index at ingest time (HashMap<String, Vec<(NodeId, u32)>>). For regex, use the `regex` crate with timeout. Cap results at `top_k` (default 50). No shell-out to ripgrep — keep it pure Rust.
